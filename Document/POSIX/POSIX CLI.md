# POSIX CLI

## POSIX란?

------

Portable Operating System Interface의 줄임말로 서로 다른 UNIX OS의 공통 API를 정리하여 이식성이 높은 유닉스 응용 프로그램을 개발하기 위한 목적으로 IEEE가 책정한 애플리케이션 인터페이스 규격이다. ([출처](https://ko.wikipedia.org/wiki/POSIX))

즉, 한 운영체제에서 개발한 프로그램을 다른 운영체제에서도 쉽게 돌아가도록 하는 것이다. Unix, macOS가 따르고 있다.

## CLI 란?

------

CLI는 Command Line Interface의 약자로 명령어를 이용해 사용자와 컴퓨터가 상호 작용하는 방식을 뜻한다.

## GUI vs CLI

------

GUI는 Graphic User Interface의 약자로 사용자가 편리하게 사용할 수 있도록 입출력 등의 기능을 알기 쉬운 아이콘 따위의 그래픽으로 나타낸 것이다.

우리가 평소 쓰는 윈도우, macOS 등이 GUI를 지원한다.

**GUI가 편리한데 왜 CLI를 배워야 할까?**

## CLI를 배워야하는 이유

------

GUI는 직관적이고 사용이 매우 쉽다. 하지만 CLI는 어렵다. 그럼에도 CLI를 배워야 하는 이유는 우리의 의사를 정확하게 전달할 수 있고, 특히 시간의 순서에 따라 명령을 내릴 수 있기 때문이다.

## 실습환경

------

윈도우 환경에서 POSIX 실습을 진행하기 위해서는 실습환경을 따로 구축해 주어야 한다.

방법은 두 가지가 있다.

- Emulater 설치
- POSIX 서버(POSIX에 호환되는 서버)를 구축 후 Secure Shell을 사용하여 원격제어
