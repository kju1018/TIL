# 기본지식

- erp: 기업 내 생산, 영업 등 경영 활동을 통합적으로 관리하는 프로그램이라고 알고있습니다.
- 리팩토링: 외부동작을 바꾸지 않으면서 내부 구조를 개선하는 방법입니다.
- 관심있는 개발언어 : 코틀린 이유: nullsafe 언어이기 때문
- 서비스를 운영하고있는 상태에 서비스가 다운 되었을때 가장 먼저 조치해야하는 것은? 서비스부터 재가동 한 후 로그 트래킹
- 세션이란: 통신을 하기 위해 서로 연결된 순간부터 통신을 마칠 때 까지의 기간을 의미한다.
- 스레드란 : 프로세스(process) 내에서 실제로 작업을 수행하는 주체를 의미합니다.

모든 프로세스에는 한 개 이상의 스레드가 존재하여 작업을 수행합니다.

- 프로세스: 실행중인 프로그램

- 커널과 쉘의 차이:

  소프트웨어-하드웨어 사이의 커뮤니케이션을 관리하는 핵심 프로그램 운영체제 내부에서 하드웨어 제어하는 역할 커널에게 명령을 전달할 수 있습니다. 사용자-운영체제 사이의 커뮤니케이션이 가능하도록 명령어를 해석해 줌

- 클라우드 컴퓨팅 : 컴퓨터를 활용하는 작업(자료 처리, 저장, 전송, 감상 등)에 필요한 데이터나 저장소 등 다양한 요소들을 인터넷 상의 서비스를 통해 제공하는 것을 의미합니다.

- OUTER JOIN, INNER JOIN 차이점

  OUTER JOIN 은

  LEFT OUTER JOIN 왼쪽 전체와 오른쪽에서 조건에 맞는것

  RIGHT OUTER JOIN 오른쪽 전체와 왼쪽에서 조건에 맞는것

  FULL OUTER JOIN 전체 합집합 리턴

  INNER JOIN : A, B의 교집합

- 스프링이란 자바 플랫폼을 위한 오픈소스 애플리케이션 프레임워크이며 동적인 웹 사이트를 개발하기 위한 여러 가지 서비스를 제공한다

- Bean클래스:  데이터를 표현하는 것을 목적으로 모든 필드는 private이며, getter/setter메서드를 통해서만 접근이 가능하다.

- POJO(Plain Old Java Object) 란 말 그대로 평범한 자바 오브젝트이다.

- **MVC패턴이란?**Model , View , Controller 의 합성어로 소프트웨어 공학에서 사용되는 소프트웨어 디자인 패턴이다.

  Model : 데이터를 처리하는 로직

  View : 사용자가 보게 될 결과 화면을 출력한다.

  Controller : 사용자의 입력처리와 흐름 제어를 담당한다.

- DI : 의존적인 객체를 직접 생성하거나 제어하는 것이 아니라, 특정 객체에 필요한 객체를 외부에서 결정해서 연결시키는 것

- http https 차이: HTTPS 프로토콜은 SSL(보안 소켓 계층)을 사용함으로써 http보다 보안이 뛰어난 프로토콜

- DAO : Data Access Object의 약자로, 데이터베이스의 데이터에 접근하기 위해 생성하는 객체이다.

- DTO :Data Transfer Object의 약자로, 계층간 데이터 교환을 위한 객체입니다.

- get post방식 차이:

  URL에 변수(데이터)를 포함시켜 요청한다. 그래서 보안에 취약하다 URL에 변수(데이터)를 노출하지 않고 요청한다. 그래서 기본 보안은 되어있다.

- 객체지향 프로그래밍이란: 프로그래밍에서 필요한 데이터를 객체로 만들고 그 객체들 간의 상호작용을 통해 로직을 구성하는 프로그래밍입니다.

- 캡슐화 : 변수와 함수를 하나의 단위로 묶는 것을 의미. 프로그램의 세부 구현을 외부로 드러나지 않도록 특정 모듈 내부로 감추는 것이다.

- 추상화: 불필요한 정보는 숨기고 중요한 정보만을 표현함으로써 프로그램을 간단히 만드는 것이다

- 상속 : 상속은 새로운 클래스가 기존의 클래스의 자료와 연산을 이용할 수 있게 하는 기능이다.

- 다형성: 하나의 객체가 여러 가지 타입을 가질 수 있는 것을 의미합니다.

- 자원을 이름(자원의 표현)으로 구분하여 해당 자원의 상태(정보)를 주고 받는 모든 것을 의미한다

- HTTP 메소드 GET, POST, PUT, DELETE

- 데이터 : 사실을 나타내는 수치 데이터 자체로는 큰 의미가 없습니다. 하지만 정보로 데이터를 가공하면 의미가 생깁니다.

- 데이터베이스의 구조나 제약조건 등을 기술한 메타데이터 집합

- 어노테이션: 프로그램에게 추가적인 정보를 제공해주는 메타데이터(meta data: 데이터를 위한 데이터)라고 볼 수 있다.

- 현실 세계의 존재하는 데이터와 그들간의 관계를 표현하기 위해서 사용하는 다이아그램입니다.
